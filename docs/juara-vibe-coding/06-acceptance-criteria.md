# 06 — Acceptance Criteria

Gunakan checklist ini sebelum deploy dan submit.

## Event Requirements

- [ ] Aplikasi aktif di Cloud Run atau Google AI Studio.
- [ ] Video demo 2–3 menit di LinkedIn.
- [ ] Post LinkedIn public.
- [ ] Menggunakan hashtag #JuaraVibeCoding.
- [ ] Link final dikirim melalui form resmi.

## Problem Score

- [ ] Landing page menjelaskan target user dalam 10 detik.
- [ ] Ada problem statement yang jelas.
- [ ] Ada pain point cards.
- [ ] Ada persona onboarding.
- [ ] Ada narasi dampak: membantu user lebih siap lapor.

## Solution Score

- [ ] Flow utama berjalan dari onboarding sampai report.
- [ ] Dashboard menampilkan SPT Readiness Score.
- [ ] Ada Tax Health Badge.
- [ ] Ada Next Best Action.
- [ ] Ada Missing Documents.
- [ ] Ada estimasi pajak dengan breakdown.
- [ ] Ada Tax Readiness Report PDF.
- [ ] Ada demo mode/sample data.
- [ ] UX mudah dipahami tanpa membaca dokumentasi.

## Uniqueness Score

- [ ] AI memberi insight personal berbasis data user.
- [ ] Ada Ask My Tax Data atau quick prompts.
- [ ] OCR memiliki manual verification.
- [ ] Ada what-if simulation.
- [ ] AI digunakan sebagai bagian dari workflow, bukan chatbot tempelan.

## Safety & Trust

- [ ] Disclaimer tampil di kalkulator.
- [ ] Disclaimer tampil di AI assistant.
- [ ] Disclaimer tampil di billing/draft payment.
- [ ] Disclaimer tampil di PDF report.
- [ ] Billing tidak mengklaim menerbitkan kode resmi.
- [ ] OCR tidak auto-save tanpa validasi user.
- [ ] AI tidak mengklaim sebagai konsultan pajak final.

## Technical Readiness

- [ ] Tidak ada error kritis di console.
- [ ] Build berhasil.
- [ ] Test tax engine utama hijau.
- [ ] Halaman utama mobile responsive.
- [ ] Demo data bisa di-reset.
- [ ] Export PDF berfungsi.
